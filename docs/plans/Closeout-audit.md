# DeerFlow 教育工作台 V1.3 封板审计

## 1. 审计结论

- 主链路代码已可运行，前端教育页与聊天增强已落地。
- `CP2/CP3` 回退与护栏逻辑在状态机层可验证通过。
- 模型实测仍有不稳定点：`Step 3.5 Flash` 偶发输出自由文本确认，未严格走 checkpoint 协议，影响“全自动一键闭环”的稳定性。

## 2. P0/P1/P2 缺口清单

### P0（已修复）

1. Clarification 选项被逐字拆分
- 现象：`options` 为字符串时被按字符编号，审批卡不可用。
- 处理：后端 middleware 增加 `options` 容错归一化（数组 / 字符串化数组 / 普通字符串）。

2. 模型不走 `ask_clarification` 时审批卡消失
- 现象：模型直接输出审批文本，前端按普通 assistant 气泡显示。
- 处理：前端消息分组增加审批文本兜底，命中 checkpoint 文本时仍渲染审批卡。

### P1（待持续优化）

1. 正常路径模型协议遵循度不稳定
- 现象：模型可能要求补充信息，但不输出标准 checkpoint 结构，导致自动化正常路径难以稳定一次性跑通。
- 建议：
  - 强化 Lead 提示词的 checkpoint 输出约束；
  - 在后端增加“自由文本确认 -> checkpoint 结构化”的中间层归一化（长期方案）。

2. 质量约束仍依赖模型遵循
- 现象：预算/安全/课时一致性已写入 SOUL/skills，但在极端回合仍可能被弱执行。
- 建议：
  - 增加 Presentation 前的硬校验器（程序化检查后再允许进入 Reviewer）。

### P2（后续优化）

1. 收尾脚本可进一步拆分为：
- 快速验收（状态机 + 合同）
- 全链路模型验收（长时运行）

2. 质量评分可升级为：
- 规则评分 + LLM 评审双轨评分，并沉淀趋势数据。
